üöÄ Quick Start

The camera position editor provides two powerful methods for finding and configuring camera positions:

Quick Access

1. Press Ctrl + E to toggle the camera editor
2. Use WASD keys to move around the scene
3. Click on objects to set look-at targets
4. Press P to print current position to console
5. Press Export Code button to get formatted output

üéØ Method 1: Built-in Position Editor

How it works:

A real-time editor that lets you fly around your scene and capture camera positions instantly.

  • Real-time positioning
  • Visual feedback with markers
  • Instant code generation
  • No external tools needed
  • Perfect for quick adjustments

‚ùå Cons:

  • Approximate positioning
  • Manual input required
  • No precision snapping

Controls:

W A S D Move horizontally

üì∑ Method 2: Embedded GLB Cameras

How it works:

Add camera objects directly in your 3D modeling software (Blender, Maya, etc.) and export them with your GLB files.

‚úÖ Pros:

  • Pixel-perfect positioning
  • Professional workflow integration
  • Visual camera setup in 3D software
  • Multiple camera angles per model
  • Version controlled with models

‚ùå Cons:

  • Requires 3D modeling knowledge
  • File re-export needed for changes
  • Slightly larger file sizes

Workflow:

1. In Blender/Maya, add a Camera object to your scene
2. Position the camera where you want the portfolio camera to be
3. Point the camera toward your desired target
4. Name the camera descriptively (e.g., "DetailView", "OverheadAngle")
5. Export GLB with camera included
6. The camera will automatically appear in the editor's "Embedded Cameras" list

üîß Integration Workflow

Complete Setup Process:

1. Design Phase: Use Blender/Maya to set up rough camera positions
2. Import Phase: Load your GLB files - embedded cameras will be detected automatically
3. Fine-tuning Phase: Use the built-in editor to adjust positions
4. Export Phase: Generate code and integrate into your main.js
5. Testing Phase: Test camera animations in your portfolio

üîç Troubleshooting

Camera not moving smoothly?

Hold Shift while using WASD for faster movement, or use smaller increments for precision.

Embedded cameras not detected?

Make sure your camera objects are actually exported with the GLB file. Check the export settings in your 3D software.

Position looks different in portfolio?

Remember that the editor uses a temporary camera mode. Test your positions by actually updating the code and reloading.

üéØ Best Practice: Combine both methods! Use embedded cameras for major positioning, then use the built-in editor for micro-adjustments and rapid iteration.
